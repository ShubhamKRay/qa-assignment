## 📄 Assignment Overview

This repository contains my submission for the **QA Engineer Intern** assignment at **SimplifyMoney**.

The focus of this assignment is to **test critical application flows** and identify **functional and real-world issues** from user interactions, ensuring reliability, security, and smooth user experience.

> **Note:** Mandatory assignment feedback from 3 users has been included in the attached PDF.

---

## 🔍 Scope of Testing

Tested core user flows include:

- **Buy Flow**  
- **Sell Flow**  
- **Gift Flow**  
- **SIP (Systematic Investment Plan) Flow**  
- **Login / Authentication Flow** (session handling, OTP, biometric, MPIN)

---

## 🧪 Testing Approach

- Manual testing of real application flows  
- Validation using **both valid and invalid inputs**  
- Edge case testing including multiple login/logout attempts and OTP behavior  
- Focused on **functional bugs**; UX/design issues noted as suggestions only  
- Repeated scenarios to ensure consistency and reproducibility  

---

## 🐞 Key Findings

- **OTP delays** during repeated login attempts  
- **Session handling issues:** automatic logout on user switch without confirmation  
- **Biometric / MPIN authentication inconsistencies**  
- Users reported **difficulty understanding app navigation and flows**

> All bugs were observed on real devices and documented with screenshots in the PDF.
